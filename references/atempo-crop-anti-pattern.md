# atempo 加速 + 裁剪反模式（致命错误）

> 所属模块：video-creator / references
> 版本：v1.0.0
> 用途：记录音频处理中的致命反模式，避免音画不同步

---

## 核心问题

**绝对禁止**：先生成超长配音 → atempo 加速 → 裁剪到目标时长

**后果**：音频时间轴与字幕完全不匹配，音画严重不同步。这是本次 session 发现的致命问题。

---

## 错误流程（导致音画不同步）

```
1. 生成84秒配音（内容完整但太长）
2. atempo 1.2x → 70秒
3. 裁剪到59秒
4. 基于59秒时长生成字幕 → 字幕与实际内容完全对不上
```

**实测结果**：
- 84秒配音 × 1.2x atempo → 70秒
- 裁剪到59秒
- 字幕时间轴完全错位（字幕从0秒开始，但音频内容已被裁剪移位）
- 用户听到的词和看到的字幕完全不匹配

---

## 正确流程（音频字幕同步）

```
1. 确定目标时长（如59秒）
2. 根据目标时长编写精简配音文本（如51秒内容 = 59秒配音）
3. 生成精确时长的配音
4. apad/silence 补齐到目标时长（如51秒→59秒）
5. 基于配音文稿（不是时长）生成字幕时间轴
```

**关键原则**：
- 字幕时间轴必须基于配音文稿的语义内容分配，不是基于音频时长推算
- 如果配音比目标时长短，用 apad/silence 填充末尾（不是裁剪音频）
- 如果配音比目标时长长，重新编写更短的配音文本

---

## 验证方法

```bash
# 检查音频时长（应该是目标时长）
ffprobe -v error -show_entries format=duration -of default=noprint_wrappers=1 audio/final.m4a

# 检查字幕时间轴覆盖范围
grep "Dialogue:" audio/subtitles.ass | tail -5

# 验证音画同步：播放测试（必须人工验证）
mpv out/final_with_subs.mp4
```

---

## 关键教训

| 反模式 | 正确做法 |
|--------|----------|
| 长音频→atempo→裁剪 | 根据目标时长编写精确配音文本 |
| 基于时长生成字幕 | 基于配音文稿语义分配字幕时间轴 |
| 裁剪音频 | apad/silence 补齐（保持原始时间轴） |
