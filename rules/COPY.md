# 营销文案生成规范

> 所属模块：video-creator / SKILL.md → 文案生成
>
> 生成两份文件：`docs/copy.md`（三版文案合集）、`docs/wechat-copy.md`（公众号专属文案）

---

## 一、通用规范（所有文案共用）

### 1.1 标题规范

| 规则 | 要求 |
|------|------|
| 字数 | **8-32字** |
| 风格 | 吸引眼球、有悬念、数字冲击 |
| 可用符号 | 感叹号、问号、省略号 |

### 1.2 摘要规范

| 用途 | 要求 |
|------|------|
| 短文案摘要 | 不超过 **30字**，一句话核心卖点 |
| 公众号 frontmatter summary | **100-200字**，概括全文核心价值 |
| 小红书摘要 | 不超过 **44字**，突出吸引点 |

### 1.3 标签规范（强制）

| 平台 | 标签数量 | 格式 | 示例 |
|------|----------|------|------|
| 小红书 | **不少于5个** | `#标签1 #标签2 #标签3` | `#AI #开源 #工具 #科技 #效率` |
| 公众号 | **不少于5个** | frontmatter tags 列表 + 正文末尾 | `#AI #开源 #工具 #科技 #效率` |
| 视频号 | 建议3-10个 | 放在正文末尾或描述中 | 结合内容关键词 |
| 抖音 | 建议3-10个 | `#标签1 #标签2 #标签3` | `#AI #工具 #教程` |

标签必须满足以下三类：
- ✅ **行业标签**（如 #AI、#开源、#编程）
- ✅ **应用场景标签**（如 #效率、#开发、#学习）
- ✅ **工具/技术标签**（如 #机器学习、#时序预测）

### 1.4 正文结构规范（通用骨架）

```
┌─ 开头（hook）     用痛点或共鸣开场
├─ 引出问题         阐述问题本质
├─ 解决方案         介绍工具/方法
├─ 功能介绍         列出核心功能（精简，3-5 点）
├─ 效果展示         展示实际效果/数据
├─ 行动号召（CTA）  引导安装/使用
└─ 结尾             总结 + 引导关注
```

### 1.5 视觉分隔规范

```markdown
段落间用空行分隔

---
重要转折用分隔线
```

## 二、文件一：`docs/copy.md` — 三版文案合集

生成要求：包含**短文案**、**中文案**、**长文案**三个版本，每个版本都必须带标签列表。

### 2.1 短文案模板（30字以内）

适用于：视频号/抖音短视频标题、朋友圈、社群分享

````markdown
## 短文案

[一句话核心卖点，不超过30字]

---

**标签**: #标签1 #标签2 #标签3
````

### 2.2 中文案模板（300-500字）

适用于：公众号推文（此处为精简预览，详细公众号格式见第三章）、小红书笔记

**小红书版**：

```markdown
## 中文案（小红书）

### [标题：8-32字，带 emoji]

[用痛点或共鸣开场，1-2 句话]

[引出问题：为什么这件事很重要]

[解决方案：介绍工具/方法，3-5 点核心功能]

[效果展示：用户能获得什么价值]

[行动号召]

---

**标签**: #标签1 #标签2 #标签3 #标签4 #标签5
```

**公众号版**（完整格式见第三章 `docs/wechat-copy.md` 规范）。

### 2.3 长文案模板（落地页/文章阅读页）

适用于：`docs/landing-page.html`、`docs/article-page.html`

```markdown
## 长文案（落地页）

### [主标题：数字冲击型，8-16字]

[副标题：补充说明核心价值，不超过30字]

> [核心引语/数据亮点]

### 痛点共鸣

[目标用户当前面临的问题，2-3 个场景]

### 解决方案

[一句话概括：什么工具解决什么问题]

| 功能 | 说明 | 价值 |
|------|------|------|
| 功能1 | 简要说明 | 用户收益 |
| 功能2 | 简要说明 | 用户收益 |
| 功能3 | 简要说明 | 用户收益 |

### 效果展示

- **量化成果**：[数据提升/时间节省等]
- **真实案例**：[用户故事或场景]

### 为什么选择 [工具名]

1. **[核心优势1]** — [一句话论证]
2. **[核心优势2]** — [一句话论证]
3. **[核心优势3]** — [一句话论证]

### 立即开始

[行动号召：安装命令 / 访问链接 / 扫码]

---

**标签**: #标签1 #标签2 #标签3 #标签4 #标签5
```

## 三、文件二：`docs/wechat-copy.md` — 公众号文案

### 3.1 公众号平台要求

1. **Frontmatter 字段**（必须）：
   - `title`: 文章标题（8-32字）
   - `summary`: 文章摘要（100-200字）
   - `tags`: 标签列表（**不少于5个**）
   - `source`: 来源链接（如有）
   - `date`: 发布日期

2. **正文结构**：
   - 以 `# 标题` 开头
   - 正文段落精简有力
   - 以 `---` 分隔线结束正文
   - 以 `**标签**:` 结尾
   - 正文末尾附来源/源码链接（如有）

3. **链接处理**：
   - 外部来源 → frontmatter 的 `source` 字段记录
   - GitHub/GitCode 源码 → 正文末尾 `# 源码地址`
   - 其他来源 → 正文末尾 `# 原文地址`

### 3.2 公众号完整模板

```markdown
---
title: [8-32字，吸引眼球]
summary: [100-200字，概括全文核心价值]
tags:
  - [行业标签]
  - [场景标签]
  - [工具标签]
  - [补充标签1]
  - [补充标签2]
source: [来源链接，可选]
date: [YYYY-MM-DD]
---

# [标题：与 frontmatter title 一致]

> [开篇引语/核心数据，可选]

[用痛点或共鸣开场，1-2 段]

## [引出问题]

[为什么这件事值得关注，1-2 段]

## [解决方案]

[介绍工具/方法，重点讲 what + why]

### [核心功能 1]

[一句话说明]

### [核心功能 2]

[一句话说明]

### [核心功能 3]

[一句话说明]

## [效果/价值]

[用户能获得什么实际收益]

---

**标签**: #标签1 #标签2 #标签3 #标签4 #标签5

# 原文地址
[来源链接]
```

### 3.3 公众号文案禁用语

- ❌ 公众号封面图建议
- ❌ 最佳发布时间
- ❌ 发布指南（这些在 `docs/posting-guide.md` 中）
- ❌ 冗长的功能列表（精简为核心 3-5 点）
