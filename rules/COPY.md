# 营销文案集生成规范

> 所属模块：video-creator / SKILL.md → 文案生成

## 输出格式

生成 `docs/copy.md`，包含**短文案**、**中文案（公众号/小红书）**、**长文案（落地页）**三版本，**每个平台版本必须包含标签列表**。

## 标签规范（强制）

| 平台 | 标签数量 | 格式 | 示例 |
|------|----------|------|------|
| 小红书 | **不少于5个** | `#标签1 #标签2 #标签3` | `#AI #开源 #工具 #科技 #效率` |
| 视频号 | 建议3-10个 | 放在正文末尾或描述中 | 结合内容关键词 |
| 抖音 | 建议3-10个 | `#标签1 #标签2 #标签3` | `#AI #工具 #教程` |

**标签要求**：
- 必须与内容主题相关
- 至少包含1个行业标签（如 #AI、#开源、#编程）
- 至少包含1个应用场景标签（如 #效率、#开发、#学习）
- 至少包含1个工具/技术标签（如 #机器学习、#时序预测）

# 工具推荐 | 让你的 AI 成为知识管家

> 「你知道吗？LLM 每次回答问题都在重复同样的工作。」

## 你是否也有这样的困扰？

每次向 AI 提问，它都要从海量文档中重新检索、重新推导。

- RAG 检索 → 片段拼接 → 重新推导
- 问一个需要综合 5 份文档的问题，AI 每次都要「从零开始」
- **没有知识积累，只有重复劳动**

## 解决方案：LLM Wiki

**Obsidian LLM Wiki** 采用了 Andrej Karpathy 的 LLM Wiki 理念：

LLM **持续构建和维护一个持久化的 wiki**——知识编译一次，然后**保持最新**，而非每次查询重新推导。

## 三层架构设计

| 层级 | 目录 | 说明 |
|------|------|------|
| Schema | CLAUDE.md | 配置层，告诉 LLM 如何工作 |
| Wiki | wiki/ | 可写层，LLM 生成和维护 |
| Raw | raw/ | 不可变层，原始资料（只读）|

### 安装使用

1. 在 Claude Code 中输入 `/obsidian-llm-wiki`
2. Skill 会引导你初始化目录结构
3. 运行 `/obsidian-llm-wiki ingest` 处理资料

- 字数：**8-32字**
- 风格：吸引眼球、有悬念、数字冲击

## 摘要规范

- 字数：不超过44字
- 内容：概括核心价值

# 公众号文案生成规范

> 所属模块：video-creator / SKILL.md → 文案生成

## 公众号官方要求

公众号文案必须遵循以下格式要求：

1. **Frontmatter 字段**（必须）：
   - `title`: 文章标题
   - `summary`: 文章摘要（100-200字）
   - `tags`: 标签列表（**不少于5个**）
   - `source`: 来源链接（如有）

2. **正文结构**：
   - 以 `# 标题` 开头
   - 以 `## 正文` 或直接段落开始正文
   - 以 `---` 分隔线结束正文
   - 以 `**标签**:` 结尾（标签用 `#` 格式，用空格分隔）

3. **链接处理**：
   - 如果内容来自链接，在 frontmatter 的 `source` 字段记录
   - 或在正文末尾以标题展示：
     - **GitHub / GitCode / GitLab 等源码地址** → `# 源码地址`
     - **其他来源** → `# 原文地址`

## 输出格式

生成 `docs/wechat-copy.md`，Frontmatter + 正文格式：

```markdown
---
title: 效率提升20倍！内容创业者的神器
summary: 不是才华，而是精力。80%时间花在琐碎工作上？baoyu-skills帮你翻转这个比例。
tags:
  - 标签1
  - 标签2
  - 标签3
  - 标签4
  - 标签5
source: https://example.com/original-link
---

# 效率提升20倍！内容创业者的神器

正文内容...

---

**标签**: #标签1 #标签2 #标签3 #标签4 #标签5

# 源码地址（GitHub/GitCode等源码）
https://example.com/repo
```

## 标签规范（强制）

公众号文案：
- **必须**在 frontmatter 的 `tags` 字段中包含**不少于5个标签**
- **必须**在正文末尾以 `**标签**: #标签1 #标签2 #标签3 #标签4 #标签5` 格式结尾
- 标签必须与内容主题相关
- 至少包含1个行业标签（如 #AI、#开源、#编程）
- 至少包含1个应用场景标签（如 #效率、#开发、#学习）
- 至少包含1个工具/技术标签

## 去掉无用信息

公众号文案**不要包含**以下内容：
- ❌ "公众号封面图建议"
- ❌ "最佳发布时间"
- ❌ "发布指南"
- ❌ 冗长的功能列表（精简为主）

## 正确的正文结构

```markdown
# 标题（吸引眼球）

正文内容，精简有力

---

**标签**: #标签1 #标签2 #标签3 #标签4 #标签5

# 原文链接（如果有）
https://example.com/original-link
```
